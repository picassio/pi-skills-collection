# Capacitor iOS Integration for Three.js Apps

Deploy Three.js web apps as native iOS apps using Capacitor.

---

## Philosophy: Two Runtimes, One Contract

Treat the project as two systems that must agree:
- A **web renderer runtime** (Three.js + Vite)
- A **native runtime wrapper** (Capacitor iOS)

Most failures happen when their contract is implicit.
Make file paths, animation names, build output, and iOS package manager choices explicit.

**Before implementing, ask:**
- What is the exact web output directory (`dist` or `www`) and does Capacitor `webDir` match it?
- Is iOS using SPM or CocoaPods, and are plugin dependencies compatible?
- Are desktop and touch controls intentionally mapped, or left to defaults?

## Quick Start: SPM Workflow

Capacitor 8+ supports Swift Package Manager by default.

### One-time Setup

```bash
npm install @capacitor/core@latest
npm install -D @capacitor/cli@latest @capacitor/ios@latest

# Build web assets first
npm run build

# Add iOS with SPM (preferred)
npx cap add ios --packagemanager SPM
npx cap sync ios
```

### Day-to-day Loop

```bash
npm run build
npx cap sync ios
npx cap run ios
```

Or open in Xcode:
```bash
npx cap open ios
```

### Simulator Tips

```bash
# List available targets
npx cap run ios --list

# Run on specific simulator
npx cap run ios --target <TARGET_ID>
```

## Project Structure

Prefer this shape for minimal ambiguity:

```
project/
├── index.html
├── src/
├── public/
│   └── assets/          # GLBs, JSON contracts, textures
│       └── assets_index.json
├── capacitor.config.ts  # webDir: "dist"
├── ios/                 # Generated by Capacitor
└── dist/                # Vite build output
```

**Asset Loading**: Use absolute URLs for files under `public/assets`:
```javascript
// ✓ Works in both browser and WKWebView
fetch('/assets/assets_index.json')

// ❌ Avoid filesystem paths or env-specific base URLs
fetch('./assets/assets_index.json')  // May break in Capacitor
```

## Controls: Desktop and Touch

Set `OrbitControls` mappings explicitly for consistent behavior:

```javascript
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;
controls.dampingFactor = 0.05;

// Explicit mouse mappings
controls.mouseButtons = {
    LEFT: THREE.MOUSE.ROTATE,
    MIDDLE: THREE.MOUSE.DOLLY,
    RIGHT: THREE.MOUSE.PAN
};

// Explicit touch mappings
controls.touches = {
    ONE: THREE.TOUCH.ROTATE,
    TWO: THREE.TOUCH.DOLLY_PAN
};
```

**If product requires vertical-only pan**, constrain after `controls.update()` each frame.

## Animation Contract via `assets_index.json`

Use one source of truth for animation metadata:
1. Load index JSON
2. Load skeleton GLB and animation GLB
3. Resolve each UI button to a clip by `sourceClipName`
4. Build `AnimationAction` map keyed by app id
5. Play default action from index

**Required startup assertions:**
- Index contains target character
- `animations[]` is non-empty
- Every `sourceClipName` resolves to a clip
- Default animation id exists (or safe fallback)

Fail loudly with clear error messages when the contract is invalid.

## Migrating from CocoaPods to SPM

Two options:

1. **Recreate** iOS platform with SPM:
   ```bash
   # Back up ios/ if needed
   rm -rf ios/
   npx cap add ios --packagemanager SPM
   npx cap sync ios
   ```

2. **Use migration helper**:
   ```bash
   npx cap spm-migration-assistant
   ```

### Validation

```bash
npx cap doctor
```

Look for:
- Matching `@capacitor/*` versions
- iOS status healthy
- `Package.swift` present for plugins

## Common Gotchas

### "It works in browser but not simulator"

**Causes**: Stale web assets, wrong `webDir`, bad asset paths.

**Fix**:
1. `npm run build`
2. `npx cap sync ios`
3. Confirm `capacitor.config.*` uses `webDir: 'dist'`
4. Use `/assets/...` URLs for files under `public/assets`

### "Animation button does nothing"

**Causes**: `sourceClipName` mismatch, clip in different GLB.

**Fix**: Log available clips at startup and compare exact string names.

### "Pan behavior feels wrong on touch"

**Causes**: Default OrbitControls mappings don't match UX.

**Fix**: Set both `mouseButtons` and `touches` explicitly.

### "Capacitor asks for CocoaPods"

**Causes**: Project created on older template.

**Fix**: Use Capacitor 8+ with `npx cap add ios --packagemanager SPM`.

### "Xcode project confusion"

- CocoaPods → uses `.xcworkspace`
- SPM → uses `.xcodeproj` with package dependencies
- Always use `npx cap open ios` to open the right project

## Performance on iOS

- Cap pixel ratio: `Math.min(devicePixelRatio, 2)` (critical on Retina iPads)
- Reuse AnimationMixer/actions; do not recreate per interaction
- On resize, always update camera aspect, projection, and renderer size
- Keep animation switching with fade transitions from metadata defaults
- Test on real device — simulator GPU behavior differs from Metal

## iOS Configuration

For app permissions and capabilities:
- Edit `ios/App/App/Info.plist`
- Configure Signing & Capabilities in Xcode
- Use official Capacitor iOS configuration docs as source of truth

---

## Anti-Patterns

❌ **Running iOS without rebuilding web assets** — simulator shows stale JS/CSS

❌ **Mixing SPM and CocoaPods** — dependency drift and broken Xcode expectations

❌ **Hardcoding clip names in UI handlers** — a renamed clip silently breaks buttons

❌ **Leaving control mappings implicit** — desktop and mobile behavior diverge

❌ **Debugging native first for web contract errors** — check JSON keys and clip resolution first
